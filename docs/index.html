atmux

Run multiple Claude Code agents in a single tmux session. Add agents, point them at repos, and monitor everything from a live dashboard.

$ brew install juanheyns/tmux/atmux

What is atmux?

atmux is a lightweight orchestration shell for Claude Code. It manages multiple Claude instances inside tmux, each pointed at a repository. A curses-based dashboard shows real-time status, and hooks track whether each agent is idle, busy, or running subagents.

Multi-agent

Spin up as many Claude instances as you need, each in its own tmux window with its own repo.

Live dashboard

Real-time overview with status indicators, subagent counts, and an interactive command bar.

Workspace state

Agent configs persist across sessions. Stop and restart without losing your setup.

Extensible widgets

Drop executable scripts in .atmux/widgets/ to add custom sections to the dashboard.

Installation

Homebrew (recommended)

brew install juanheyns/tmux/atmux

Manual install

curl -fsSL https://raw.githubusercontent.com/juanheyns/agent-tmux/main/install.sh | bash

From source

git clone https://github.com/juanheyns/agent-tmux.git
cd agent-tmux
./install.sh

Requirements

  • tmux
  • Claude Code (claude CLI)
  • Python 3.10+
  • gh CLI (optional, for cloning with owner/repo shorthand)

Quick start

# Initialize a workspace
cd ~/projects/my-workspace
atmux init

# Add agents
atmux add backend owner/api-service          # clone from GitHub
atmux add frontend https://gh.corp/ui.git    # full URL
atmux add infra ./terraform                   # local directory

# Launch
atmux start

# Later — agents persist across sessions
atmux stop
atmux start   # all agents restored

Custom Claude flags

Pass extra flags to Claude on a per-agent basis. Flags after -- are persisted and restored on start.

atmux add dev1 owner/repo -- --model sonnet --max-turns 50
atmux add reviewer owner/repo -- --permission-mode default

Commands

CLI

atmux init                              # Initialize workspace
atmux start                             # Launch session (default)
atmux stop                              # Stop all agents
atmux add <name> <repo> [-- flags]      # Add an agent
atmux remove <name|all>                 # Remove agent(s)
atmux send <name|all> <msg>             # Send message to agent(s)
atmux list                              # List configured agents
atmux heartbeat <name|all> <s> <msg>   # Set heartbeat
atmux heartbeat stop <name|all>         # Stop heartbeat
atmux heartbeat list                    # List heartbeats
atmux shell <dir>                       # Open a shell window

Dashboard commands

Type these in the dashboard's command bar:

add <name> <repo> [-- flags]          # Add agent
send <name> <msg>                     # Send message
remove <name>                         # Remove agent
heartbeat <name|all> <s> <msg>       # Set heartbeat
heartbeat stop <name|all>             # Stop heartbeat
stop                                  # Stop everything

tmux navigation

Ctrl-a n/p    # Next/previous window
Ctrl-a <n>    # Jump to window by number (0=overview, 1=first agent, ...)
Ctrl-a d      # Detach (agents keep running)

Widgets

Extend the dashboard with custom scripts. Drop an executable in .atmux/widgets/ and it appears as a section.

# .atmux/widgets/git-status
#!/usr/bin/env bash
if [[ "$1" == "--setup" ]]; then
  echo '{"title": "Git Status", "interval": 15}'
  exit 0
fi

for dir in "$ATMUX_WORKSPACE"/agents/*/; do
  name=$(basename "$dir")
  branch=$(git -C "$dir" branch --show-current 2>/dev/null)
  echo "$name: $branch"
done

Widget protocol

  • --setup — Return JSON with title and interval (seconds). Optional; defaults to filename and 10s.
  • Normal run — Print lines to stdout. Each line becomes a row in the dashboard section.
  • Environment — ATMUX_WORKSPACE and ATMUX_DIR are available.
  • Timeout — Scripts that take >2s are automatically backed off to 3x their runtime.

Workspace layout

.atmux/
  config.json       # Session name, workspace settings
  agents.json       # Persisted agent list
  heartbeats.json   # Heartbeat configs
  status/           # Runtime state (ephemeral)
  widgets/          # Custom dashboard scripts
agents/
  backend/          # Cloned repos live here
  frontend/

Standalone scripts

These work from any terminal in the workspace directory (or any subfolder) — you don't need to be inside the tmux session.

send <name|all> <msg>    # Send a message to an agent (or all)
list                     # List configured agents
stop                     # Stop all agents and kill the session

Debugging

Set ATMUX_DEBUG=1 in the agent's environment to enable hook logging to .atmux/status/hooks.log.

# In your atmux start, or export before launching:
export ATMUX_DEBUG=1
atmux start

# Watch the log
tail -f .atmux/status/hooks.log
